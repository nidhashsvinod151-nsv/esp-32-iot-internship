Zelbytes IoT Platform — Architecture Preview

## Overview
Zelbytes is building a dedicated IoT learning platform at careers.zelbytes.com
with fleet HTTP ingest and an MQTT broker for intern capstones. Integration
is not live for this cohort yet. This document studies the planned
architecture, drafts client config stubs, and maps the Day 8 SensorBundle
schema to anticipated future API fields, so switching from the lab Mosquitto
broker to production is a config change, not a rewrite.

Planned endpoints (subject to change):
- `POST https://careers.zelbytes.com/api/v1/iot/telemetry`
- `mqtts://careers.zelbytes.com:8883`
- Device tokens issued from the careers portal

## Authentication Model (Planned)

Zelbytes plans per-device authentication using a token issued from the
careers portal — likely a JWT or static API token. Expected usage:
- HTTP: sent as `Authorization: Bearer <DEVICE_TOKEN>` header
- MQTTS: sent as MQTT username/password or client certificate at CONNECT

Exact mechanism (JWT vs static token) is not yet finalized.
